2. Kiến trúc graph

Workflow chuẩn production cho agent xử lý ticket hỗ trợ: chuẩn hoá input → phân loại bằng LLM (structured output) → định tuyến theo state → gọi mock tool khi cần → LLM-as-judge đánh giá kết quả tool → retry có giới hạn → cổng phê duyệt HITL cho hành động rủi ro → mọi nhánh kết thúc tại finalize → END.

8. Extension đã làm

  • RAG grounding — knowledge base TF-IDF thuần Python, answer_node truy xuất top-k tài liệu để grounded; make grade-rag pass 10/10 (top-1 doc đúng + keyword + forbidden).
  • SQLite persistence — SqliteSaver + WAL mode, thread_id riêng mỗi scenario; xác minh có 12 checkpoint và get_state_history() truy xuất được.
  • Real HITL interrupt — approval_node hỗ trợ LANGGRAPH_INTERRUPT=true dùng interrupt(), mặc định mock approval an toàn cho CI.
  • LLM-as-judge — evaluate_node dùng LLM chấm chất lượng kết quả tool, có guard theo schema.
  • Mermaid graph diagram — sơ đồ kiến trúc trong report & trang này.
